# Recording-analyzer statistics verification

reca.sh uses ffmpeg to analyze and generate statistical information about a music file. The verification tools in this directory analyze the same audio file with different tools such as SoX and Python.

Although results ideally would be identical, each tool uses slightly different algorithms, so exact agreement isn't expected. verify.sh uses independent tools to check whether ffmpeg's output is reasonable. A default threshold of 10% is defined for each statistic generated by recording-analyzer. Custom thresholds can be set for each statistical metric and can be edited by the user. Setting a threshold to 0% would mean that the statistic from ffmpeg must exactly match the statistic from the other program. Thresholds are set in percentages, e.g. 5% (see "Setting Thresholds" below.)

Running the verification program is optional and not part of reca.sh.

## 📦 Installation

Download the zip file from <https://github.com/mcochris/Recording-analyzer>

or

```bash
git clone https://github.com/mcochris/Recording-analyzer.git
```

## ✅ Requirements

The following programs need to be in the $PATH variable and executable from the command line:

- BASH shell
- reca.sh
- ffmpeg
- jq
- SoX, plus its libsox-fmt-all library
- Python version 3, plus its soundfile and numpy libraries
- loudgain
- bs1770gain
- ebumeter

## ▶️ Usage

```bash
cd path/to/recording-analyzer/verification/
./verify.sh <audio_file>
```

Example:

```bash
cd path/to/recording-analyzer/verification/
./verify.sh ~/Music/aimee\ mann\ -\ Voices\ Carry\ \(Single\ Mix\).mp3
```

## 📏 Setting Thresholds

There is a threshold.txt text file in each verification subdirectory. The file contains one number between 0% and 100%. Edit this file to change the percentage of acceptable difference between the ffmpeg number and the alternative tool's number. If the threshold.txt file is missing or invalid, the default percentage is set to 10%.
