Recording-analyzer statistics verification
==========================================

The recording-analyzer program uses the ffmpeg program to analyze and generate statistical information about a music file. The programs in this verification directory can analyze the same audio file with different programs such as SoX and the soundfile module for the Python v3 programming language.

Ideally, running a different program on the same audio file should produce the same statistical output as the recording-analyzer program. That would help the user verify that the results produced by the recording-analyzer program are accurate. Unfortunately, different programs analyze audio in different ways than ffmpeg and may produce different results. A statistical threshold is used to check if the statistical results from ffmpeg are within a tolerance of other programs. A default threshold is defined for each statistic generated by the recording-analyzer program. The thresholds can be edited by the user. Setting a threshold to 0 would mean that the statistic from ffmpeg much exactly match the statistic from the other program. Setting the threshold to .01 would mean the statistics would have to be within 1% of each other.






ffmpeg: https://www.ffmpeg.org/
SoX: https://sourceforge.net/projects/sox/
Python soundfile: https://python-soundfile.readthedocs.io/
